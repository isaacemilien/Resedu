Week 9: HTML Forms and Styling

Basics of creating and styling forms on a webpage.

By Isaac Emilien

October 29, 2022

This week’s learning was about the implementation and styling of forms, consisting of uses for forms, form elements, and basic form CSS.

HTML Form Uses

Forms are used as a gateway for information to be collected from users, finding applications in account utilities, and customer service. Fully functional forms however typically make use of client side scripting language for validation, and a server side scripting language for data exchange.

HTML Form Elements

Core HTML forms are composed of two elements: form and input. The form element acts as a container for all inner elements that will make up a form. Input elements make up the fields for which users interact witn.

Implementations

While implementing this week’s HTML forms, for the contact page, there were issues trying to use HTML built in validation. From reading Mozilla’s HTML documentation on input [8] attributes it was realised that an input element needs to have the ‘required’ field attached in order to properly use validation.

Week 8: Responsive Web Design Layouts

Deeper look at placing objects on a page responsively

By Isaac Emilien

November 18, 2022

This week knowledge was gained about CSS layout properties, use of layout modules, and media query breakpoints

CSS Layout Properties

CSS layout properties define how HTML elements should be placed. Layout properties include float, clear, and position. Float pushes an element to the bounds of its container in a given direction. Clear defines the point at which an object should be moved to the row below. Position overrides the positioning constraints of the container it is in.

CSS Media Query Breakpoints

CSS media query breakpoints define a viewport value at which a style should be enacted.

Implementations

While implementing CSS media queries for different parts of the learning journal it seemed as though CSS was being duplicated.

With this, further research was done on responsive design using W3CSchools [7]. From this it was understood that CSS implemented thus far was not mobile first, and had been designed with desktop in mind. To solve this problem default styles were tailored for mobile, allowing for removal of the two duplicate points.

Week 7: Further responsive web design

Takes deeper look at responsive web design elements

By Isaac Emilien

November 11, 2022

CI 435 week 7 further explored principles of responsive web design, topics covered were: how to approach the implementation of responsive design elements, making media responsive, and the safe use of media from the web.

Core Principles

The core principles of how to approach responsive web design implementation is the 3 column method, this proposes that the structure of web pages when viewed on mobile take up 1 column, on tablet or mobile portrait take up 2 columns, and on desktop take up 3 columns.

Responsive Media

Responsive media is the sizing of webpage images and videos based on screen size, which can be challenging factoring the range of media quality that can be chosen. Media flexibility using vanilla CSS can be achieved using media queries and ratio based width height sizes.

Safe Media

Safe use of media is of high priority given our web pages are available to the open web, meaning even though we are in an educational context, the content we use is still open to the public, leaving us liable. To account for this creative commons content has be used.

Week 6: Introduction to Responsive Web Design

Covers starting points for creating responsive web pages

By Isaac Emilien

November 5, 2022

This week knowledge gained covered responsive web design using fluid layouts, and media queries, and responsive media.

CSS Fluid Layouts

Fluid layouts use relative units and CSS layout modules to create responsive content. CSS layout modules such as grids, flexbox, and multi-column, allow for row-column placing of elements without having to use the position properties directly.

CSS Media Queries

Media queries are CSS elements that apply styles to HTML based on certain units, in our case the size of the screen; this allows us to structure our webpage to best suit the screen size of the device that it is being viewed on.

Week 5: CSS Box Model and Relative Styles

How the CSS block model and relative styles can be used to better position and size elements

By Isaac Emilien

October 29, 2022

This week CSS knowledge was further improved, covering, positioning and sizing elements using the box model, and styling text.

Box Model

The CSS box model allows one to dictate how parts of html are shown and placed on a page. In the box model parts of HTML are seen as boxes, these boxes can be logically moved or placed inside the bounds of other elements.

Relative Units

Relative units in CSS are values that change based on the page. This allows values that are assigned relative units to be dynamic to the size of content.

Implementations

This week fallback fonts were added, allowing unavailability of fonts to be accounted for and hinder user performance less. After a fallback font was selected, the custom font Lato was also added; to do so a new font file was added to the website folder structure, and a font face was defined in the styles.css. During implementation of the font there were legal and ethical concerns, this is because implementing external content made by someone without their permission could breach intellectual property rights, resulting in potential liability. To account for this the font Lato was sourced from Google Fonts [6], which is free to use, open source.

CSS

CSS is a markup language that stands for Cascading Style Sheets. CSS is used to customise the presentation of HTML. [5]

Week 3: Further HTML Document Structure

A deeper look at laying out a web page with HTML

By Isaac Emilien

October 15, 2022

This week’s learning covered the use of section and div elements, navigation, and various elements to display information.

Section and Div Elements

Section and div elements are used to define logical collections of information on the page. Sections more so encompass typical page specific content such as headers, footers, and navigation. Div elements define looser general divisions of content.

Document Navigation

Document navigation is achieved using anchors and links. Anchors are defined using the 'a' tag; anchors hold links which connect to another page, or page area.

Week 2: HTML Markup basics

Simple websites structuring using HTML

By Isaac Emilien

October 8, 2022

This week learning covered the basics of HTML which included: tags and elements, block and inline elements, and natural document structure. Furthermore, newly learned elements were implemented.

Tags and Elements

Tags and elements are keywords in HTML that when applied to text define their meaning [2]. Elements are the function itself, tags define bounds of an element.

Block and Inline Elements

HTML block and inline are two types of element. Block elements naturally stack horizontally and fill a page row, examples include h1, and p. Inline elements are held inside block elements and can be used to perform a function on them.

Natural Document Structure

HTML documents naturally follow a nested structure enabling:

  • Placement of elements within or below one another
  • Logical placement of content on page

This week's lectures covered the fundamentals of web development. This covers responsive web design, standards, and reasoning behind HTML and CSS rules. Further knowledge gained about these were:

Responsive design is the adjusting of website scaling based on the device screen. Meaning that the content can be seen on a large variety of devices.
